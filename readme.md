# Платформа TeamFinder

**TeamFinder** — это веб-платформа для поиска единомышленников и формирования команд для pet-проектов. Специалисты могут публиковать свои идеи, находить команду и управлять своими профилями.

## Реализация Варианта 2: «Навыки» и фильтрация участников

В соответствии с выбранным вариантом, в проекте реализован следующий функционал:

### 1. Управление навыками (Skills) в профиле
*   **Отображение:** В публичном профиле пользователя отображается блок «Навыки» в виде тегов.
*   **Интерактивность (AJAX):** Владелец профиля может добавлять и удалять навыки без перезагрузки страницы.
*   **Автодополнение:** При добавлении реализован живой поиск по существующей базе навыков с подсказками (до 10 совпадений). Если навыка нет в базе, его можно создать прямо из интерфейса.

### 2. Фильтрация участников по навыкам
*   **Панель фильтрации:** На странице «Участники» добавлена визуальная панель со списком всех существующих в системе навыков.
*   **Логика:** Клик по тегу применяет фильтр и отображает только тех пользователей, которые владеют данным навыком.
*   **UX:** Выбранный навык подсвечивается в списке, предусмотрена ссылка «Сбросить» для возврата к полному списку участников.

### 3. Общий функционал платформы
*   **Custom User Model:** Реализована кастомная модель пользователя с авторизацией по Email.
*   **Автоматические аватары:** При регистрации генерируется уникальный аватар на основе первой буквы имени с использованием библиотеки `Pillow`.
*   **Валидация:** Реализована проверка номеров телефонов (уникальность и формат) и ссылок на GitHub (валидность домена).
*   **Проекты:** Реализован полный цикл управления проектами (создание, редактирование, изменение статуса «Открыт/Закрыт», участие в чужих проектах).

## Технологический стек
*   **Backend:** Python 3.13, Django 5.2, PostgreSQL
*   **Frontend:** HTML5, CSS3, JavaScript (Fetch API / AJAX)
*   **Инфраструктура:** Docker Compose (PostgreSQL)

## Инструкция по запуску

### 1. Подготовка окружения
Создайте виртуальное окружение и установите зависимости:

```bash
python -m venv venv
venv\Scripts\activate     # Для Windows
source venv/bin/activate  # Для Linux/Mac
pip install -r requirements.txt
```

### 2. Настройка проекта
Создайте файл .env на основе примера .env_example.
Запустите базу данных через Docker:
```bash
docker compose up -d
```
### 3. Инициализация базы данных и запуск
```bash
python manage.py migrate
python manage.py createsuperuser
python manage.py runserver
```
Проект будет доступен по адресу: http://localhost:8000
